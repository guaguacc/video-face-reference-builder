# Video Face Reference Builder 架构设计

## 1. 总体目标

项目目标是从非配合人脸视频中提取可追溯的人脸参考信息。系统不直接承诺真实还原完整正脸，而是先构建一套证据链：

```text
视频证据 -> 候选关键帧 -> 原始关键帧证据集 -> 可选增强对照 -> 最完整原始面部基准帧 -> 局部人脸参考 -> 候选完整脸校正依据
```

第一阶段最关键的是“关键帧选择”。如果关键帧选错，后续 CodeFormer 辅助增强、脸型估计、局部校正都会建立在错误证据上。

## 2. 第一阶段核心流程：关键帧选择

关键帧选择不是简单均匀抽帧，也不是预先指定时间窗口。它应采用：

```text
全局检索 + AI 判断 + 时间窗口细查 + 输出原始关键帧
```

完整结构如下：

```text
输入视频
  ↓
A. 全局候选抽帧
  ↓
B. 基础算法评分
  ↓
C. 生成全局 contact sheet
  ↓
D. AI 视觉判断候选帧
  ↓
E. 聚类出关键时间窗口
  ↓
F. 窗口内加密抽帧
  ↓
G. 输出最终关键帧
  ↓
H. 可选 CodeFormer 辅助增强
  ↓
I. AI 局部裁剪 + 标点
  ↓
J. AI 整体审核 Review Sheet
  ↓
K. OpenCV 局部对齐与拼接候选
```

## 3. 各阶段职责

### A. 全局候选抽帧

系统必须先覆盖完整视频，不能直接从已知时间窗口开始。

推荐策略：

- 60 秒以内视频：每 `0.5s-1.0s` 抽一张
- 60 秒到 5 分钟：每 `1.0s-2.0s` 抽一张
- 更长视频：先每 `2.0s-5.0s` 抽一张，再根据候选结果自适应加密

输出：

```text
keyframe_selection/global_candidates/t_00038.500.jpg
```

### B. 基础算法评分

基础评分用于稳定排序和过滤，不作为唯一决策。

第一版评分字段：

- 清晰度：Laplacian variance
- 亮度/曝光：过暗和过曝扣分
- 运动模糊代理指标
- 中心区域信息量
- 肤色区域线索
- 重复帧相似度
- 人脸或局部脸检测置信度，后续接入
- 透视/裁切风险，后续接入

不能只按全图清晰度排序。全图清晰度容易把背景边缘、手部、字幕、压缩噪声误判成高质量帧。

### C. 全局 Contact Sheet

系统把全局候选帧拼成 contact sheet，供 AI 视觉判断和人工复核。

输出：

```text
keyframe_selection/contact_sheets/global_candidates_001.jpg
```

### D. AI 视觉判断候选帧

AI 判断每张候选帧是否具有人脸参考价值。

AI 输出应为结构化 JSON，字段包括：

```json
{
  "timestamp": 38.5,
  "face_reference_value": 85,
  "visible_regions": ["mouth", "philtrum", "nose"],
  "problems": ["partial_face", "minor_motion_blur"],
  "reason": "Upper lip, philtrum, and lower nose are visible with usable shading."
}
```

AI 负责语义判断，基础算法负责稳定评分和全局覆盖。

### E. 聚类关键时间窗口

系统根据全局候选结果聚类出关键时间窗口。

窗口必须从全局检索结果中发现，不能预先写死。

示例：

```text
38.0s, 38.5s, 39.0s, 39.5s -> 37.5s-40.0s
48.0s, 48.5s, 49.0s, 49.5s, 50.0s -> 47.5s-50.5s
```

如果某些单点候选不属于高分窗口，但包含独特信息，例如下颌线、侧脸、额头、肤色，也应保留为 singleton candidate。

### F. 窗口内加密抽帧

在关键窗口内做更密集的时间采样。

推荐策略：

```text
0.1s-0.25s 抽一张
```

这一步用于找到同一动作或姿态中的最佳瞬间。

输出：

```text
keyframe_selection/refined_windows/window_01/t_00038.250.jpg
```

### G. 最终关键帧输出

最终输出只以原始视频帧作为第一阶段主结果，包含原始帧、评分和选择原因。

目录结构：

```text
keyframe_selection/
  global_candidates/
  refined_windows/
  selected_keyframes/
  contact_sheets/
    global_candidates.jpg
    refined_candidates.jpg
    selected_keyframes.jpg
  ai_global_judgement.json
  keyframe_selection.json
  keyframe_selection.md
```

### H. 可选 CodeFormer 辅助增强

CodeFormer 作为关键帧之后的辅助分支，而不是关键帧筛选器。

规则：

- 输入只能来自 `selected_keyframes/`，不直接处理整段视频。
- 原始关键帧永远保留为主证据。
- 增强成功的图进入 `enhanced_optional/`，只作为清晰度参考或对照材料。
- 增强失败的图不删除、不降权，对应原始关键帧仍继续用于后续重建。
- 增强图不得覆盖同名原图，也不得替换 `selected_keyframes.jpg` 的结论。

输出：

```text
keyframe_selection/
  selected_keyframes/
  selected_keyframes_final/
    original/
    enhanced_optional/
  contact_sheets/
    selected_keyframes.jpg
    codeformer_optional_success.jpg
    codeformer_original_only.jpg
  codeformer_auxiliary_summary.json
  codeformer_auxiliary_summary.md
```

如果 16 张关键帧中只有 2 张增强成功，最终仍然是：

```text
16 张原始关键帧主证据 + 2 张可选增强参考
```

### I. AI 局部裁剪 + 标点

AI 对 `selected_keyframes/` 做区域级裁剪时，必须同时输出局部关键点，而不是只输出 bbox。

输出字段：

```json
{
  "source": "selected_003_t_00039.000.jpg",
  "region": "mouth",
  "bbox": [0, 850, 390, 1160],
  "points": {
    "left_mouth_corner": [48, 178],
    "right_mouth_corner": [318, 154],
    "upper_lip_center": [178, 123],
    "lower_lip_center": [188, 214],
    "philtrum_center": [172, 62]
  },
  "quality": {
    "usable_for_alignment": true,
    "occlusion": "low",
    "blur": "medium",
    "perspective_risk": "high"
  }
}
```

坐标约定：

- `bbox` 使用原始关键帧坐标。
- `points` 使用 crop 内局部坐标。
- 系统必须保留 crop 到原图的 offset，便于追溯和回投影。

不同区域最少标点：

- mouth：左右嘴角、上唇中心、下唇中心、人中中心。
- nose/philtrum：鼻尖、左右鼻翼/鼻孔、人中中心、上唇中心。
- eye：内眼角、外眼角、上眼睑中心、下眼睑中心。
- face_shape：脸颊边缘、下颌/下巴候选点；如果不可信应标记为低可信，不强行拼接。

### J. AI 整体审核 Review Sheet

系统生成带点位的 review sheet，由 AI 进行整体审核，不要求人工确认。

审核目标：

- 点位是否落在正确语义位置。
- bbox 是否包含足够上下文。
- 是否存在遮挡、水印、悬浮按钮干扰。
- 是否可用于 OpenCV 对齐。

审核输出：

```json
{
  "accepted": ["mouth:selected_003_t_00039.000"],
  "rejected": ["mouth:selected_014_t_00038.500"],
  "warnings": ["nose crops are high perspective risk"],
  "next_stage_inputs": ["accepted alignment crops only"]
}
```

### K. OpenCV 局部对齐与拼接候选

OpenCV 只使用通过 AI 审核的点位和 crop，不直接用粗裁剪图做整脸逆透视。

拼接基准必须来自当前视频中最完整的原始关键帧，不能使用 AI 生成的 full face reference 作为基准。AI 生成图只能作为后续候选输出或对照，不得进入证据基准链。

推荐策略：

- 2-3 个稳定点：使用 `cv2.estimateAffinePartial2D` 做局部仿射。
- 4 个以上可靠点：局部区域可尝试 `cv2.getPerspectiveTransform`。
- 拼接融合：优先使用 alpha feathering；边界明显时使用 `cv2.seamlessClone`。
- 不做整脸 homography；当前素材近距离畸变严重，整脸逆透视不可信。

### L. 强特征包驱动的 AI 完整脸候选

当 OpenCV 局部拼合效果差，且用户明确要求生成完整脸候选时，进入 AI 复合分支。

该分支的目标不是美化最完整原图，也不是让模型自由生成一张干净自拍，而是：

1. 用最完整的原始关键帧确定脸型、近距离透视、肤色、头发边界和器官大致位置。
2. 从已裁剪的局部参考中选出少量最可信切片，组成强特征参考包。
3. 用 `imagegen` skill 生成完整脸候选，并在 prompt 中把嘴、鼻、人中、眼、脸颊分别写成硬约束。
4. 输出候选图和“特征包 vs 候选图”的 review sheet，由 AI 整体审核局部特征是否被保留。

候选 04 使用的记录文件：

```text
docs/candidate_04_feature_driven_flow.md
```

候选 04 的关键经验：

- 大 contact sheet 作为参考太弱，模型容易生成泛化人脸。
- 必须把嘴、鼻、人中、眼、脸型切片单独放大成 feature pack。
- prompt 必须明确“先放置局部器官，再保守推断缺失半边脸”。
- 输出完整脸后，仍应继续做局部 masked edit；一次性生成整脸会让头发、眼睛和缺失轮廓被 AI 推断过多。

## 4. 评分融合

最终选择不只依赖 AI，也不只依赖算法评分。

建议第一版融合公式：

```text
final_score =
  deterministic_score * 0.55
  + ai_reference_value * 0.35
  + diversity_bonus * 0.10
```

其中：

- `deterministic_score`：清晰度、曝光、中心信息量等代码评分
- `ai_reference_value`：AI 对人脸参考价值的判断
- `diversity_bonus`：用于避免最终帧全部集中在同一个区域，例如全是嘴部

权重必须写入报告，便于后续调整。

## 5. 旧九张帧的定位

旧案例中的九张帧：

```text
38.0s, 38.5s, 39.0s, 39.5s,
48.0s, 48.5s, 49.0s, 49.5s, 50.0s
```

是人工观察全局预览后选出的关键时间点，不是模型自动选出的结果。

它们只能作为 benchmark：

- 检查新流程是否能从全局检索中发现这些窗口
- 检查新流程是否能找到比它们更好的帧
- 检查 AI 是否漏掉有价值的嘴部、人中、脸型信息

不得把这些时间点硬编码进项目逻辑。

## 6. 第一版实践范围

第一版实践先实现：

- 全局候选抽帧
- 基础算法评分
- contact sheet 生成
- 时间窗口聚类
- 窗口内加密抽帧
- 可选 CodeFormer 辅助增强汇总
- AI 局部裁剪、标点和整体审核 review sheet
- 结果报告

AI 视觉判断可以先以结构化 JSON 文件接入：

- 第一轮可以由人工/模型填写 `ai_global_judgement.json`

后续再接入自动视觉模型调用。

## 7. 通用化约束

项目流程必须适用于不同脸部视频，不能围绕当前案例硬编码。

必须参数化的内容：

- 全局采样间隔
- 最大候选帧数量
- AI 判断文件路径
- 时间窗口聚类间隔
- 窗口加密采样间隔
- 最终关键帧数量
- CodeFormer 路径和辅助增强开关
- 最低 AI 参考价值阈值

不得硬编码：

- 当前案例的视频文件名
- 当前案例的 `38s-40s` 或 `47s-50s` 时间窗口
- 当前案例的九张旧参考帧
- 当前人物的脸部特征

当前案例只能作为 benchmark，用于检查通用流程能否从全局检索中发现高价值片段。

## 8. 实践中暴露的问题

当前案例实践证明：

- 只按全图清晰度排序会偏向手臂、背景边缘和高反差区域。
- 只看 top-score contact sheet 会隐藏后半段有价值的人脸局部信息。
- 时间窗口不能按时间顺序截断，必须按候选综合分排序。
- 使用 AI 判断时，未标注帧不能给中性高分，否则遮挡帧会混进最终结果。
- 第一阶段必须保留原始关键帧作为主证据，不把增强或生成结果混入关键帧选择结论。
- CodeFormer 对近距离局部脸检测失败很常见；失败只代表增强分支不可用，不代表原始关键帧不可用。
- AI 裁剪必须配套标点和审核；只裁 bbox 不能支撑可靠拼接。

这些问题应进入测试和报告逻辑，而不是靠人工记忆规避。
