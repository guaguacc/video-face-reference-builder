# Video Face Reference Builder 实施计划

## 1. 第一阶段目标

建立一个可复现的本地 Python 管线，能对案例视频完成：

- 读取视频信息
- 抽取采样帧
- 计算基础质量分
- 筛选最佳帧
- 生成参考板
- 生成处理报告

第一阶段先不自动生成完整正脸，也不训练模型。

## 2. 第一阶段项目结构

```text
Video Face Reference Builder/
  README.md
  项目大纲.md
  需求规格.md
  实施计划.md
  pyproject.toml
  src/
    vfrb/

  tests/

    test_reference_boards.py

```

## 3. 技术约束

- 使用 Python。
- 图像/视频处理优先使用 OpenCV 和 Pillow。
- 不覆盖原始视频、原始帧和已有案例结果。
- 输出目录按每次运行独立创建。
- 测试优先覆盖可确定逻辑，视频模型类能力后续再加集成测试。

## 4. 任务拆分

### 任务 1：项目骨架与测试入口

创建 Python 包、测试目录、运行脚本和基础配置。

验收：

- 可以运行测试命令。
- `src/vfrb` 可以被导入。

### 任务 2：视频信息读取

实现 `video_info.py`。

输入：

- 本地视频路径

输出：

- 文件名
- 分辨率
- 帧率
- 总帧数
- 时长
- 编码信息尽量读取

验收：

- 能读取当前案例视频。
- 缺失文件时报清晰错误。

### 任务 3：抽帧

实现 `extract_frames.py`。

能力：

- 支持按间隔抽帧
- 支持最大帧数限制
- 稳定命名输出帧

验收：

- 输出 `frames/frame_000001.jpg` 这类文件。
- 不覆盖已有目录，或明确清理临时运行目录。

### 任务 4：帧质量评分

实现 `score_frames.py`。

第一版评分：

- Laplacian variance 清晰度
- 平均亮度
- 过暗/过曝惩罚
- 边缘风险占位字段
- 透视风险占位字段

验收：

- 输出 CSV 或 JSON 评分表。
- 可按综合分排序选出最佳帧。

### 任务 5：参考板生成

实现 `build_reference_boards.py`。

第一版生成：

- 全局最佳帧参考板
- 清晰度最高帧参考板
- 亮度合理帧参考板

后续再扩展：

- 嘴部参考板
- 人中参考板
- 鼻部参考板
- 脸型参考板

验收：

- 输出 JPEG 参考板。
- 图片尺寸稳定，标题和文件名可读。

### 任务 6：报告输出

实现 `generate_report.py`。

报告包含：

- 输入视频信息
- 参数
- 抽帧数量
- 最佳帧列表
- 输出文件路径
- 当前未实现能力说明
- 透视/镜头畸变/脸型风险字段

验收：

- 输出 `report.md`。
- 能说明哪些结果来自真实视频帧，哪些还没有进行 AI 补全。

### 任务 7：端到端脚本

实现 `pipeline.py` 和 `scripts/run_case.py`。

验收：

- 一条命令可以处理当前案例视频。
- 结果输出到 `outputs/<run_id>/`。

## 5. 暂缓内容

以下内容放到第二阶段：

- 自动人脸关键点检测
- 嘴部/人中/鼻部区域裁切
- 脸型轮廓估计
- 近距离透视校正
- 候选完整脸局部校正
- 身份相似度评分

CodeFormer/GFPGAN 这类单帧增强工具只作为关键帧之后的可选辅助分支，不进入关键帧筛选主流程。增强成功的图作为对照参考，增强失败时继续使用原始关键帧。

## 6. 第一阶段完成标准

对当前案例视频运行后，项目应生成：

```text
outputs/<run_id>/
  video_info.json
  frames/
  frame_scores.json
  best_frames/
  reference_boards/
  report.md
```

并且测试命令通过。
